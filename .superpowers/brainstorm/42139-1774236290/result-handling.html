转换成功后：如何呈现结果？

服务端返回转换后的文件下载 URL，用户接下来如何操作？

A

转换完成弹窗 + 分享/保存

转换完成后弹出成功提示卡片，显示文件名和大小，提供两个按钮：「保存到文件」和「分享」（调 UIActivityViewController）

优点

  • 操作最直接，无需额外跳转
  • 保存到文件/AirDrop/微信等均支持

缺点

  • 无法在 App 内预览文件内容
B

先下载到临时目录 → 用 QLPreviewController 预览 → 再保存/分享

转换完成后，先下载到本地缓存，弹出系统预览（Quick Look），用户在预览界面可点击右上角分享/保存

优点

  • 用户可以确认文件内容再决定保存
  • 用系统原生预览，体验一致

缺点

  • 多一步下载；PDF/图片效果好，但 Word/Excel 在 QL 内可能排版有出入
C

直接自动下载并保存到「文件」App

转换完成后静默下载，自动保存到 App 沙盒/用户选定的目录，弹出"已保存"Toast 提示

优点

  • 操作最少，体验最流畅

缺点

  • 用户无法选择保存位置；文件管理混乱
  • 无法立即确认内容
