完整设计总结

所有决策汇总，确认后写入 Spec 文档

交互流程（已确认）

有格式选项的工具（PDF转换器 / Word / Excel / PPT 转换）
点击卡片 → 自定义底部面板（列表行样式选格式）→ UIDocumentPickerViewController 选文件 → push ConvertJobViewController → 开始转换 → QLPreviewController
无格式选项的工具（PDF转Word ⭐ / 文件转图片）
点击卡片 → UIDocumentPickerViewController 选文件 → push ConvertJobViewController → 开始转换 → QLPreviewController
合并 PDF（单入口，自动判断接口）
点击卡片 → UIDocumentPickerViewController（多选）→ push ConvertJobViewController（显示文件列表）→ 开始转换 → 自动判断调 mergetwopdf/mergemorepdf → QLPreviewController

新增组件

ConvertAPIService.swift
  • multipart/form-data 上传
  • URLSession 上传进度回调
  • 解析 newslist → 下载 URL
  • 下载结果文件到 temp 目录
  • 统一错误处理（复用 APIError）
ConvertJobViewController.swift
  • 4状态UI：idle / fileSelected / converting / success+error
  • 两阶段进度条：上传 + 处理中
  • 合并PDF 支持文件列表
  • 成功后 present QLPreviewController
  • MeshGradient 背景 + 颜色系统
FormatPickerSheet（内嵌在 ConvertViewController）
  • 自定义底部面板，随 dim 遮罩弹出
  • 列表行：彩色图标 + 格式名 + 扩展名说明
  • 卡片风格与 ToolCell 一致
  • 取消按钮
  • 支持深/浅色模式

ConvertViewController 改动

  • 给 ConvertToolItem 增加 kind: ConvertToolKind 枚举，定义每个工具的 API 和文件类型
  • didSelectItemAt：有格式选项 → 先 present FormatPickerSheet；无格式选项 → 直接打开文件选择器
  • 格式和文件选好后，push ConvertJobViewController
